CYB Build

Nonostante il nome fuorviante, si tratta di un processore di task generici. Per task si intende una generica operazione esterna al programma, ossia un qualunque processo.

Il programma permette di definire un template di task (chiamato poco intelligentemente process, nel seguito continuerò a chiamarlo template perchè più chiaro), scegliendo tra vari task predefiniti (es git clone, msbuild, VB build ecc...). Una volta definito il template , si può generare un'istanza di esso, che opportunamente configurato può essere fatto girare: in tal caso i task verranno eseguiti uno ad uno, se uno di essi dovesse fallire il processo si interrompe. In una finestra di log viene mostrato l'output dei processi che vengono eseguiti.

I file di configurazione possono essere salvati e ricaricati: è necessario però che il template a cui un dato file di configurazione è associato sia presente, altrimenti l'importazione fallisce.

Lo scopo principale di questo pogramma è quello di creare una sequenza di operazioni per la creazione di una Build del Cybertronic, ovvero la creazione automatica dei pacchetti con i sorgenti e i binari di tutte le applicazioni coinvolte.

Creazione e configurazione del task

Una volta definito il template possiamo istanziare un processo associato ad esso: si può fare dal menu File -> New Process:

sono creati i due task del tipo definito in precedenza, che andranno poi configurati al momento dell'istanziazione di un nuovo processo. Le definizioni dei template vengono salvate in un file xml, TaskProcessDefs.xml

In questo caso il processo è terminato correttamente. I task portati a compimento vengono impostati in stato Done. Nel caso in cui uno dei task risulti errato, l'esecuzione in sequenza si interrompe e il task errato viene segnalato:

In questo caso ad esempio il task 2 contiene un path errato al file della soluzione. Correggendo l'errore e clickando su Continue l'esecuzione riparte dal task errato (per la precisione, l'esecuzione skippa i task che risultano in stato Done, ovvero eseguiti correttamente).

Da notare che lo stato dei task può essere settato in Todo o Done usando il menu a tendina sui singoli task; è possibile fare anche una selezione multipla dei task (CTRL + click) e impostare lo stato di tutti i task selezionati. Questo è utile ad esempio nel caso di processi contenenti molti task, per eseguirne solo un sotto gruppo, basta impostare a Todo i task da eseguire e in Done quelli da skippare.

XCopy

VB task

Esegue la build di un progetto VB6

PSexec

Esegue un programma su una macchina remota utilizzando il programma psexec.exe

Envelope

Si tratta di un tipo di task contenente una lista di task: Clickando su Elenco task si apre una finestra per la definizione dell'elenco dei task: I tre task predefiniti eseguono in sequenza:

  • copia dei file da cryptare da locale alla directory condivisa
  • esecuzione del comando remoto per la cryptazione dei file di output del cybertronic C#
  • copia dei file criptati dalla directory condivisa in locale

Es: cryptazione dei file di output della build di Cybertronic C#

ZIP

Es: zippa la directory con i sorgenti C#
